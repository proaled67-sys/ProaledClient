# BestClient Authored Functions Scope

This file defines the exact permission-required scope referenced by the root `LICENSE`.

## Permission-required function scope

1. All function/method implementations that define, read, write, synchronize, or apply BestClient settings listed in the `bc_*` settings registry below, including the component-mask settings consumed by the Components editor.
2. All function/method implementations in `src/game/client/components/bestclient/`, except Chat Bubbles implementation/config handling.
3. All function/method implementations in:
- `src/game/client/components/menu_media_background.h`
- `src/game/client/components/menu_media_background.cpp`
- `src/game/client/components/media_decoder.h`
- `src/game/client/components/media_decoder.cpp`
4. All function/method implementations in `bestclient-voice-server/src/main.rs`.

### Explicitly excluded from the permission-required scope

- Chat Bubbles implementation and related config/menu handling, including `bc_chat_bubbles*` and `bc_chat_bubble_*` settings.
- `src/game/client/components/bestclient/chat_bubbles.h`
- `src/game/client/components/bestclient/chat_bubbles.cpp`

## Notes

- Some BestClient-adjacent settings use non-`bc_*` prefixes; they are still covered whenever their implementations live inside the listed BestClient source paths.
- The `bc_disabled_components_mask_lo` and `bc_disabled_components_mask_hi` settings are used by the Components editor to stage and apply module visibility.

## Not in this scope

- Upstream DDNet/Teeworlds code remains under upstream licenses.
- Third-party code, assets, fonts, and libraries remain under their own licenses.
- Content in `data/` remains under per-file/per-directory licenses (including CC-BY-SA where stated).

## Maintenance rule

- When adding/removing `bc_*` settings, update this file in the same commit.
- If a BestProject-authored feature is implemented outside the listed paths, add that path in the same commit.
